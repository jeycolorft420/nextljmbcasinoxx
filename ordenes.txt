El error **"Minified React error \#130"** es muy específico: significa que React intentó renderizar un componente, pero recibió `undefined`.

En tu caso, la cadena de errores indica que **`DiceDuel.tsx` está intentando renderizar `<ThreeDDice />`, pero `ThreeDDice` no se está importando correctamente**, probablemente debido a cómo Next.js maneja los archivos CSS en la carpeta `app`.

Tienes un problema de **Importación de CSS Global en un Componente**. En Next.js (App Router), **no puedes** importar archivos `.css` normales (como `import "./ThreeDDice.css"`) dentro de un componente. Eso rompe el módulo y hace que el componente sea `undefined` en producción.

Aquí tienes la solución paso a paso para arreglarlo definitivamente:

### PASO 1: Arreglar el CSS de los Dados

El archivo `ThreeDDice.css` está causando el conflicto. Vamos a mover sus estilos a un lugar seguro o convertirlo en módulo. La forma más rápida y segura ahora mismo es:

1.  Abre `src/modules/games/dice/components/dice/ThreeDDice.tsx`.
2.  **BORRA** la línea: `import "./ThreeDDice.css";`
3.  Guarda el archivo.

Ahora, necesitamos que esos estilos existan. Tienes dos opciones (elige la **Opción A** que es la más rápida):

**Opción A (Rápida):**
Copia todo el contenido de `src/modules/games/dice/components/dice/ThreeDDice.css` y pégalo al final de tu archivo `src/app/globals.css`.

**Opción B (Limpia - Recomendada si sabes usar CSS Modules):**

1.  Renombra `ThreeDDice.css` a `ThreeDDice.module.css`.
2.  En `ThreeDDice.tsx`, cambia el import a: `import styles from "./ThreeDDice.module.css";`
3.  Tendrías que cambiar todos los `className="cube"` por `className={styles.cube}`. **(Si no quieres refactorizar todo el HTML, usa la Opción A)**.

### PASO 3: Limpieza Profunda y Reconstrucción (Obligatorio)

Este error se queda "pegado" en la caché de Next.js. Debes ejecutar estos comandos en tu servidor (la terminal negra de Antigravity o SSH):

```bash
# 1. Detener el servidor actual (Ctrl+C si está corriendo manual, o pm2 stop)
pm2 stop all

# 2. Ir a la carpeta del proyecto
cd /var/www/nextljmbcasinoxx

# 3. ¡IMPORTANTE! Borrar la caché corrupta
rm -rf .next
rm -rf node_modules/.cache

# 4. Reconstruir el proyecto limpio
npm run build

# 5. Reiniciar el backend de juegos también (por si acaso)
cd game-server
npm run build
cd ..

# 6. Arrancar todo de nuevo
pm2 restart all
```

### ¿Por qué pasó esto?

Al hacer `git pull` y `npm run build`, Next.js intentó compilar el archivo `ThreeDDice.tsx`. Al encontrar un `import "./ThreeDDice.css"`, el compilador de producción falló silenciosamente o creó un módulo vacío para `ThreeDDice` porque los archivos CSS globales solo se permiten en `layout.tsx`. Cuando `DiceDuel` intenta usar `ThreeDDice`, este es `undefined`, y React explota con el error \#130.

Haciendo el **PASO 1 (Quitar el import CSS)** y el **PASO 3 (Borrar .next)**, tu juego volverá a la vida.
